Sensitive Data Exposure
  • When any application or API do not protect sensitive data properly like user account details, credit card details, passwords etc.
  • Attacker can view this sensitive data and get access to internal network
  • GitHub Tokens and API Keys Leakages
  • Sensitive Invoices Indexed by Google
  • Insecure storage of Data also leads to SDE.

Why it Happens
  • No proper access control over when exchanging sensitive data
  • API's not properly protected
  • Lack of Robots.txt
  • Lack of Data Strip when saving at server.

(Tip: Robots.txt is a file which contains set of rules for crawlers i.e what part of website can be crawled and what not can be crawled)

What can be achieved by it
  • Sensitive Information of the Users leak, i.e -Transaction Details, Passwords
  • API Keys
  • Tokens of Internal Portals only accessible to Domain Admins
  • Passwords, SSH Keys, aka anything that is sensitive

How do we fix SDE issues
  • Don't store sensitive data unnecessarily
  • Make sure to encrypt all SD at rest
  • Disable catching Responses that has SD
  • Do not put keys, tokens in Gitub or anywhere in server.
  • Implement Access Controlfor users (Make checker Functionality)
