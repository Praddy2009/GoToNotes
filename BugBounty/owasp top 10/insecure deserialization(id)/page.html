Insecure Deserialization
  • Applications those serialisehostile or tampered object by attacker
  • When an application serialises and deserialises the data based on input without checking it properly may lead to RCE

Why it Happens
  • Accept serialized objects from untrusted source and deserialize it

What can be achieved by it
  • RCE
  • Sensitive file disclosure(etc/passwd)

how do we fix ID
  • Best way is not to acceptserialized objcts from untrusted sources
  • Integrity checksfor serialized objects to prevent hostile object creation or data tampering
  • Loggingdeserialization exceptions and failure, such as where the incoming type is not the expected type, or the deserialization throws exception
