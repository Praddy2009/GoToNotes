Data type specifies the type of data that will be stored in the column.
  • CHAR(n)
    • Fixed length alphanumaric, 1 char (min) , 2000 char (max)
    • If string of smaller length is stored, it is padded with spaces at the end
  • VARCHAR(n)/VARCHAR2(N)
    • Variable length alphanumaric, 1 char (min) , 4000 char (max)
    • If data is smaller than the specified size,only the data value is stored, no padding is done
  • DATE
    • Stores date and time, January 1 4712 B.C. (min) , December 31, 9999 (max)
    • Default format DD-MON-YY also DD-MON-YYYY
  • Number(precision, scale)/NUMERIC(p,d)
    • Floating point number/ Integer number
    • Precision is total no of significant digit (between 1 to 38 ). Scale is the number of digit to the right of the decimal point
    • If neither precision nor scale is specified the stored up to 38 digits
  • INTEGER
    • Integer Number
  • LONG
    • Stores vriable length character string containing up to 2GB
    • Similar to Varchar
    • Can be only one long data type per table
  • RAW
    • Stores binary data such as digitized picture or image. 2000 bytes max
  • LONG RAW
    • Higher range of RAW. 2GB max
    • Can be only one long Raw data type per table
  • LOB(Large Object)
    • Stores large volume of data
    • BLOB: Binary data such as audio video clips up to 4GB
    • CLOB: Character data upto 4GB
    • BFILE: Stores references to a binary file that is external to database and is maintained by OS
