Normalization
It's the process of decomposing or breaking a relation schema R into fragments (i.e. smaller schemas) R1, R2, ... Rn such that the following conditions :
  • Lossless Decomposition : The fragment should contain the same information as the original relation
  • Dependency preservation : All the functional dependencies should be preserved within each fragment Ri
  • Good form: Each fragment Ri should be free from any type of redundancy.

Decomposition
It's replaces a relation with a collection of smaller relations.
It breaks the table into multiple tables in the database.
It should always be lossless.
The decomposition of a base relation is said to be lossless if the original relation can be recovered back by joining the fragment relations.
At the time of decomposition the smaller relations must have a foreign key relation(so that natural join is possible)
