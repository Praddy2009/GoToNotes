Key
It's a property of entity set which allows us to identify a set of attributes that suffice(is adequate) to distinguish entities from each other.
  • Super Key: It's a group of single or multiple keys which identifies rows in a table. It may have additional attributes that are not needed for unique identification.
  • Here EmpSSN, EmpNum and EmpName are superkeys
  • Primary Key: A column or group of column in a table which help us to uniquely identify every row in that table is called a primary key.
  • Candidate Key: A superkey with no repeated attributes is called candidate key.
  • In the above example StudID, Roll No Email are candidate keys
  • Foreign Key:
    A foreign key is a column which is added to create a relationship with another table.
  • A FOREIGN KEY is a field (or collection of fields) in one table that refers to the PRIMARY KEY in another table.

>Compound Key: It has many fields which allow us to uniquely recognize a specific record. It's possible that each column may not be unique by itself within the database.However, when combined with the other column or columns the combination of comosite keys become unique.

Here combination of OrderNo and ProductID can serve as Compound key


>Composite Key:
A key which has multiple attributes to uniquely identify rows in a table is called a composite key. The difference between compound and the composite key is that any part of the compound key can be a foreign key, but the composite key may or maybe not a part of the foreign key.


>Surrogate Key: An artificial key which aims to uniquely identify each record is called a surrogate key.
  • These kind of key are unique because they are created when you don't have any natural primary key. They do not lend any meaning to the data in the table. Surrogate
