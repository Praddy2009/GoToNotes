Relational Model
  • It consists of a collection of tables, each of which is assigned a unique name.
  • Degree: is the number of attributes in the relation/table
  • Cardinality: is the number of tuples/rows in the relation/table.
  • Each attribute of a relation has a unique name.

Constraints
  • Every relation has some conditions that must hold for it to be a valid relation.These conditions are called Relational Intigrity Constraints.
    • Key Contraint:
      • In a relationship whith a key attribute,no two tuples can have identical values for key attributes.
      • A key attribute can not have NULL values
    • Domain Constraint:
      • In specifies that within each tuple, the value of each attribute A must be an atomic value from the domain dom(A).
    • Refrential Integrity Contraint:
      • The rule state that if a foreign key in one relation, either each foreign key value must match a primary key value in the other table or else the foreign key value must be NULL.
      • A foreign key that refrences its own relation is known as recursive foreign key.
    • Operational Constraint:
      • These are the constraints enforced in the database by the business rules or real world limitation.
