Security Misconfiguration
  • Usage of Default Creds
  • Attackers often tend to attack unpatched flaws
  • Misconfigured webservers or DNS Configurations
  • Misconfigured Access Control

Why it Happens
  • Unnecessary features enabled or installed
  • The security setting in app servers(Struts, Spring, ASP.net,libraries, DB etc)not set to secure.

What can be achieved by it
  • Account Takeovers
  • RCE
  • Malware Injections
  • Email Spoofing(Missconfigured TXT DNS Server )
  • Server Injection

How do we fix SM issues
  • Never keep the default configuration and go live
  • Update all software and applications
  • Disable all unnecessary Functions
