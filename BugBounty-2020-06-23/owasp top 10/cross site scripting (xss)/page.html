Cross Site Scripting (XSS)
  • The attack allows attacker to execute javascript
  • There are 3 typesof XSS
    • Reflected XSS
    • Stored XSS
    • DOM XSS
  • RXSS :The application includes invalidated or unescaped users input as part of HTML Output,which can be used further for ATO's(Account take overs). In case their is search bar and i input alert(1) and it gets executed then it's RXSS vulnerable.This attack, allows attacker to execute js in victims browser and hijack the session cookie leading to ATO.
  • SXSS :Which is later viewed by another user or admin(V-ATO). Their is an edit profile section and we gave js payload in firstname and lastname field and we save it. Let's say its later viewed by admin the arbitrary js payload gets executed and gives the cookies to the attacker.
  • DXSS : Client side attack, where untrusted input is passed in source and comes out of sink and executes.

Why it Happens
  • No sanitization oe Escaping
  • Input is directly taken as the HTML output
  • Bad Coding practices

What can be achieved by it
  • Account Takeovers
  • Keyloggers
  • Redirection
  • and a lot more..

how do we fix XSS
  • Input Sanitization
  • Escaping and encoding
  • WAF(to block XSS attacks)
