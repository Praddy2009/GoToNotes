Exploratory Style Vs Modern Soft. Dev


→ Use of Life Cycle Models.
⇒ Now every program is developed according to the adopted life cycle model and follows the stages as mentioned in accepted/adopted life cycle model.

→ Software is developed through several well defined stages:
⇒ Requirements analysis and specification
⇒ Design
⇒ Coding
⇒ Testing, etc.

→ Emphasis has shifted from error correction to error prevention.
⇒ Modern practices emphasize detection of error as close to their point of introduction as possible.
⇒ Focus is on detecting as many errors as possible in each phase of development.

→ In exploratory style:
⇒ Coding is synonymous with program development
→ Now:
⇒ Coding is considered only a small part of a program development effort.

→ A lot of effort and attention is now being paid to Requirements specification.

→ Also, now there is a distinct design phase.

→ During all stages of development process periodic reviews are being carried out.

→ Software testing has become systematic.Standart testing techniques are available.

→ There is better visibility of design and code:
⇒ Visibility means production of good quality, consistent and standard document.
⇒ In the past, very little attention was being given to producing good quality and consistent documents.
⇒ We will see later that increased visibility makes software project management easier.

→ Because of good documentation:
⇒ Fault diagnosis and maintenance are smoother now.

→ Several metrices are being used which help in software project management, quality assurance, etc.

→ Projects are being properly planned with proper estimation, scheduling and monitoring mechanisms.

→ Use of CASE tools.
